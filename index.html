Kom igång med Claude Code

Från nybörjare till att ha en AI-kollega som jobbar i dina filer

Workshop · 24 april 2026 · 3,5 timmar

Vad är Claude Code?

De flesta har använt AI som en avancerad sökmotor. Du ställer en fråga, du får ett svar. Claude Code är något annat. Det är mer som att ha en kollega som faktiskt jobbar i dina filer – inte bara pratar om dem.

Vanlig AI-chatt

Du skriver en fråga och får ett svar tillbaka. Du kopierar, klistrar in, justerar. AI:n ser aldrig dina filer.

Claude Code

Claude ser dina mappar och filer direkt. Den kan skapa, redigera och organisera – precis som en kollega vid datorn bredvid dig.

Varför Claude Code?

  • Arbetar direkt med mappar och filer på din dator – inget copy-paste
  • Kan skapa hela projekt med mappstrukturer, dokument och webbsidor
  • Kommer ihåg projektinstruktioner via en enkel textfil (CLAUDE.md)
  • Kan publicera ditt arbete online via GitHub
  • Kan lära sig nya beteenden via "skills" – dina egna instruktioner
  • Gratis att prova – du får ett visst antal förfrågningar utan kostnad
Tänk på det så här: Vanlig AI är som att maila en konsult och vänta på svar. Claude Code är som att ha konsulten sittande bredvid dig vid datorn, redo att faktiskt göra jobbet.

Installation

Steg för steg – du behöver inte ha gjort detta innan workshopen, men vill du förbereda dig finns allt här.

Installation på Mac

1
Skapa ett konto hos Anthropic
Gå till console.anthropic.com och skapa ett konto. Du behöver verifiera med e-post.
2
Installera Node.js
Öppna Terminal (sök efter "Terminal" i Spotlight med Cmd+Mellanslag). Klistra in:
node --version
Ser du ett versionsnummer (t.ex. v20.x.x)? Då är du klar! Om inte, gå till nodejs.org och ladda ner LTS-versionen. Installera som vanligt program.
3
Installera Claude Code
I Terminal, skriv:
npm install -g @anthropic-ai/claude-code
4
Starta Claude Code
Skapa en mapp för ditt projekt och starta Claude Code:
mkdir mitt-projekt cd mitt-projekt claude
Första gången öppnas en webbläsare för att logga in. Följ instruktionerna.
Vad är Terminal? Det är ett program som redan finns på din Mac. Istället för att klicka på ikoner skriver du kommandon. Tänk på det som att skicka textmeddelanden till din dator.

Installation på Windows

1
Skapa ett konto hos Anthropic
Gå till console.anthropic.com och skapa ett konto.
2
Installera Node.js
Gå till nodejs.org och ladda ner LTS-versionen. Installera som vanligt (klicka Next hela vägen). Öppna sedan PowerShell (sök i startmenyn) och skriv:
node --version
Ser du ett versionsnummer? Bra, då funkar det!
3
Installera Claude Code
I PowerShell, skriv:
npm install -g @anthropic-ai/claude-code
4
Starta Claude Code
Skapa en mapp och starta:
mkdir mitt-projekt cd mitt-projekt claude
Alternativ för Windows: Claude Code finns också som en desktop-app om du hellre vill slippa terminalen helt. Den fungerar likadant men med ett grafiskt fönster.

Vad är GitHub och varför behövs det?

GitHub är som en molnlagring för projekt – men med superkrafter. Det håller koll på varje ändring, låter dig publicera webbsidor gratis, och gör det enkelt att dela ditt arbete.

Du behöver GitHub för att...

  • Spara ditt arbete säkert – allt lagras online som en backup
  • Se vad som ändrats – varje ändring sparas med tidsstämpel, som "spåra ändringar" i Word
  • Publicera webbsidor – sidor du skapar med Claude Code kan publiceras live gratis
  • Samarbeta – andra kan se och bidra till ditt projekt

Skapa ett GitHub-konto

1
Registrera dig
Gå till github.com och klicka "Sign up". Gratis konto räcker.
2
Installera GitHub CLI
I terminalen:
# Mac (med Homebrew): brew install gh # Windows (i PowerShell som administratör): winget install --id GitHub.cli
3
Logga in
Skriv i terminalen:
gh auth login
Följ frågorna som dyker upp. Välj "GitHub.com" och "Login with a web browser".
Inga förkunskaper krävs. Vi går igenom allt under workshopen. Men om du vill vara förberedd sparar du ca 10 minuter genom att installera i förväg.

Övningar

Konkreta saker vi gör under workshopen – från enkla till mer avancerade

1. Hej Claude – din första konversation

Block 2 · Intro

Starta Claude Code i en tom mapp och be den presentera sig. Utforska vad den kan se och göra.

Testa:
→ "Var befinner du dig just nu? Vilka filer ser du?"
→ "Skapa en fil som heter hej.txt med en hälsning till mig"
→ "Skapa en mapp som heter dokument och lägg tre tomma filer i den"

2. Skapa en CLAUDE.md – ge Claude ett minne

Block 3 · Intro

Vi skriver en instruktionsfil tillsammans. Berätta för Claude vem du är, vad projektet handlar om, och hur du vill att det ska jobba.

Testa:
→ "Skapa en CLAUDE.md-fil för det här projektet. Jag heter [ditt namn] och jobbar med [ditt område]. Projektet handlar om [beskriv]."
→ "Jag vill att du alltid svarar på svenska och aldrig föreslår lösningar som kostar pengar utan att fråga mig först."

3. Bygg en personlig webbsida

Block 3 · Praktisk

Claude skapar en komplett webbsida åt dig med ditt namn, en kort presentation och valfri design. Du bestämmer innehållet, Claude bygger.

Testa:
→ "Skapa en enkel, snygg webbsida om mig. Jag heter [namn], jobbar som [roll] och brinner för [intressen]. Använd en modern, ljus design."
→ "Lägg till en sektion om mina projekt/erfarenheter"
→ "Ändra färgschemat till mörkblått och guld"

4. Publicera live med GitHub

Block 4 · Praktisk

Vi tar webbsidan du just skapade och publicerar den på internet – gratis, via GitHub Pages.

Testa:
→ "Hjälp mig publicera den här webbsidan på GitHub Pages. Jag vill att den ska vara tillgänglig på [dittnamn].github.io"
→ "Uppdatera sidan och publicera ändringarna"

5. Skapa din första skill

Block 5 · Praktisk

En skill talar om för Claude hur det ska hantera en specifik situation. Vi skapar en som du faktiskt har nytta av.

Exempel på skills du kan skapa:
→ "Skapa en skill som gör att du alltid levererar dokument i formatet: sammanfattning, bakgrund, rekommendation, nästa steg"
→ "Skapa en skill för kundmejl som alltid ska vara max 5 meningar, professionella men varma"
→ "Skapa en skill som genererar veckorapporter med fast mall: highlight, utmaningar, nyckeltal, plan nästa vecka"

6. Avancerat: Claude som arbetspartner

Block 5 · Fördjupning

Ge Claude ett mer komplext uppdrag och jobba iterativt – precis som med en riktig kollega.

Testa:
→ "Jag ska hålla en presentation om [ämne] nästa vecka. Hjälp mig ta fram en struktur, skapa slides-underlag och en talarmanus-outline."
→ "Skapa en intern FAQ-sida för mitt team med de 10 vanligaste frågorna om [ämne]"
→ "Analysera den här texten och ge mig tre alternativa versioner: en kort (LinkedIn), en medellång (blogg), en lång (rapport)"

Ordlista

Alla begrepp du behöver – förklarat utan krångel

Claude Code
Ett AI-verktyg som körs i terminalen (eller som app) och kan läsa, skapa och redigera filer på din dator. Till skillnad från vanlig AI-chatt kan Claude Code faktiskt göra saker – inte bara svara på frågor.
Terminal / Kommandorad
Ett program där du skriver textkommandon istället för att klicka. På Mac heter det "Terminal", på Windows "PowerShell". Tänk på det som att skicka SMS till din dator.
CLAUDE.md
En textfil som Claude läser automatiskt varje gång du startar ett projekt. Det är Claude Codes "minne" – här skriver du instruktioner, projektbeskrivning och regler. Tänk på den som en briefing till en ny kollega.
Markdown (.md)
Ett enkelt sätt att formatera text med symboler: # Rubrik, **fetstil**, - punktlista. Filändelsen .md betyder bara att filen använder denna formatering – men det är fortfarande en helt vanlig textfil som du kan öppna och redigera var som helst. CLAUDE.md, skills och andra instruktionsfiler använder markdown.
Skill
En instruktionsfil som lär Claude ett specifikt beteende. T.ex. "skriv alltid mejl i max 5 meningar" eller "följ alltid den här rapportmallen". Du skapar skills som .md-filer i en speciell mapp.
Slash command
Ett snabbkommando som börjar med /. T.ex. /help för hjälp eller /commit för att spara ändringar. Genvägar för vanliga uppgifter.
GitHub
En plattform för att lagra och dela projekt online. Fungerar som en molnlagring med "spåra ändringar"-funktion. Du kan också publicera webbsidor gratis via GitHub Pages.
GitHub Pages
En gratis funktion i GitHub som gör att du kan publicera en webbsida direkt från ditt projekt. Du får en adress som dittnamn.github.io/projektnamn.
Repository (repo)
Ett "projekt" på GitHub. En mapp med alla dina filer plus en historik över alla ändringar. Tänk: en projektmapp som aldrig glömmer.
Commit
En sparad version av ditt projekt. Varje commit är som en "checkpoint" – du kan alltid gå tillbaka. Tänk: Ctrl+S, men med en beskrivning av vad du ändrade.
Push
Att ladda upp dina sparade ändringar (commits) till GitHub. Det som ligger lokalt på din dator skickas upp till molnet.
Node.js / npm
Node.js är ett program som behövs för att köra Claude Code. npm (Node Package Manager) är verktyget som installerar Claude Code. Du behöver inte förstå mer än att det är en förutsättning som behöver finnas på datorn.
MCP (Model Context Protocol)
Ett sätt att koppla Claude Code till externa tjänster som Google Drive, Slack eller databaser. Tänk: plugins eller kopplingar. Du behöver inte veta detta dag 1, men det är nästa steg.
Behörighetsnivåer (Permission modes)
Du bestämmer hur mycket Claude Code får göra utan att fråga dig. Tre nivåer: fråga alltid (säkrast), tillåt läsning (mellanläge), full tillgång (snabbast men kräver tillit). Under workshopen kör vi på det säkraste läget.
Prompt / Instruktion
Det du skriver till Claude Code. Kan vara en fråga, ett kommando, eller en beskrivning av vad du vill ha gjort. Ju tydligare du är, desto bättre resultat.
Token / Kostnad
Claude Code mäter användning i "tokens" (ungefär ord). Ju längre konversationer och ju fler filer Claude läser, desto mer kostar det. Anthropic ger ett gratis startbelopp.
Context window (kontextfönster)
Claudes "arbetsminne" – hur mycket information den kan ha i huvudet samtidigt. Tänk: ett skrivbord med begränsad yta. Ju längre konversation och ju fler filer, desto fullare blir skrivbordet. När det blir fullt börjar Claude sammanfatta och glömma äldre delar.
/compact
Ett kommando som sammanfattar hela konversationen till en kortare version. Använd det när Claude börjar "glömma" saker du sagt tidigare – det frigör plats i kontextfönstret utan att du behöver börja om.
/clear
Rensar hela konversationen och börjar om från noll. Bra när du byter ämne helt. Claude läser fortfarande din CLAUDE.md, så projektinstruktionerna finns kvar.
Plan Mode
Ett läge där Claude visar sin plan innan den gör något. Istället för att direkt skapa filer säger den: "Jag tänker göra X, Y och Z – okej?" Du godkänner eller justerar innan något händer. Bra för att behålla kontrollen.
Modellval (Opus / Sonnet / Haiku)
Claude Code kan använda olika AI-modeller: Opus (smartast, dyrast – för komplexa uppgifter), Sonnet (balanserad – bra för vardagsarbete), Haiku (snabbast, billigast – för enkla frågor). Du kan byta under samtalets gång.
Interrupt (Escape)
Tryck Escape för att avbryta Claude mitt i en uppgift. Användbart om du ser att den är på väg åt fel håll. Inget arbete går förlorat – du kan ge nya instruktioner direkt.
Session / Konversation
Varje gång du startar Claude Code börjar en ny session. Inom en session minns Claude allt ni pratat om. Mellan sessioner börjar den om (om du inte har CLAUDE.md eller Memory). Du kan återuppta gamla sessioner med claude --resume.
Kontextröta (Context rot)
Det som händer när kontextfönstret blir för fullt: Claude tappar tråden och ger sämre svar. Lösningen är att använda /compact för att sammanfatta, eller /clear för att börja om. Ett tecken på kontextröta är att Claude upprepar sig eller "glömmer" saker du nyss sa.
Kontrollpunkter (Checkpoints)
Automatiska ögonblicksbilder som sparas innan Claude gör ändringar. Om något går fel kan du ångra med kommandot /undo. Tänk: ett automatiskt "spara innan"-system som skyddar dig.
Minne (Auto-memory)
En automatisk funktion som sparar dina preferenser och arbetsmönster över tid – mellan sessioner. Skiljer sig från CLAUDE.md: minnet byggs upp automatiskt av Claude själv, medan CLAUDE.md är instruktioner du skriver manuellt.
Multimodalt stöd
Claude kan "se" bilder. Du kan klistra in screenshots, foton av whiteboards, eller designskisser – och Claude förstår vad som visas. Användbart för att t.ex. bygga en webbsida utifrån en skiss.
Verktyg (Tools)
Inbyggda funktioner som Claude automatiskt väljer för att utföra uppgifter: läsa filer, skriva filer, köra kommandon, söka i kodbasen. Du behöver inte veta vilka verktyg som finns – Claude väljer rätt verktyg åt dig.
Utökat tänkande (Extended thinking)
En funktion som ger Claude extra tid att resonera och tänka igenom komplexa problem i flera steg innan den agerar. Aktiveras automatiskt vid svårare uppgifter. Kostar fler tokens men ger bättre resultat.

Tips och vanliga fallgropar

Gör så här

  • Var specifik. "Skapa en snygg webbsida" → "Skapa en webbsida med mitt namn, en kort bio på 3 meningar, ljus bakgrund och grön accentfärg"
  • Iterera. Du behöver inte få allt rätt första gången. Säg "ändra rubriken", "gör texten kortare", "byt färg"
  • Läs vad Claude föreslår innan du godkänner. Du har alltid sista ordet
  • Använd CLAUDE.md. Allt som Claude ska veta varje gång skriver du där
  • Spara ofta med commits. Tänk: regelbundna checkpoints

Undvik det här

  • Ladda inte upp känsliga filer (lönelistor, personnummer, kunddata) utan att först anonymisera
  • Ge inte full behörighet förrän du förstår vad Claude Code gör
  • Lita inte blint – granska alltid resultatet, särskilt siffror och fakta
  • Starta inte för stort. Ett litet projekt först, sedan bygger du vidare
Om kostnader: Claude Code använder tokens för varje interaktion. Du får ett gratis startbelopp. För att hålla kostnaderna nere: var koncis i dina instruktioner, undvik att be Claude läsa onödigt stora filer, och stäng sessionen när du är klar. Under workshopen jobbar vi med små projekt som kostar väldigt lite.
